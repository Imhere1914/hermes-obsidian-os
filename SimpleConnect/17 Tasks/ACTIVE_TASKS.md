
# Active Tasks

## Purpose

This file stores the current executable work for Simple Connect.

Tasks represent actionable operational work that moves the system toward current objectives.

Tasks should remain tightly aligned with:
- current phase
- current priorities
- current objectives
- approved direction

This file is not a brainstorming document.

---

## What Belongs Here

- active build tasks
- hardening tasks
- infrastructure tasks
- verification tasks
- dashboard tasks
- Vapi tasks
- onboarding tasks
- invoice follow-up tasks
- Discord setup tasks
- VPS tasks
- Codex tasks

---

## What Does Not Belong Here

- vague ideas
- speculative future features
- unrelated improvements
- strategic philosophy
- duplicate roadmap items
- low-priority distractions

---

## Hermes Usage

Hermes should use this file to:
- identify executable work
- determine next actions
- maintain execution continuity
- avoid wandering outside scope

Hermes should:
- prioritize highest leverage tasks
- stay within active phase boundaries
- update completed task status

Hermes should not:
- invent unrelated work
- silently expand project scope
- redefine priorities without escalation

---

## Task Structure

Each task should include:

1. Task name
2. Objective
3. Current phase
4. Priority
5. Dependencies
6. Risks
7. Verification requirement
8. Completion condition

---

## Recommended Priority Labels

- Critical
- High
- Medium
- Low
- Deferred

---

## Rule

Tasks should move the system measurably closer to:
- contractor demo readiness
- operational stability
- autonomous execution
- production readiness

If a task does not materially support those outcomes, question whether it belongs here.
