

# Master Objectives

## Purpose

This file defines the primary objectives for Simple Connect.

Objectives are not tasks.

Objectives describe the outcomes the business and system are working toward. Tasks are the specific actions required to achieve those outcomes.

This file exists so Hermes, Codex, and Dee stay aligned on what matters most and avoid drifting into busy work.

---

## What Belongs Here

- business objectives
- product objectives
- demo objectives
- revenue objectives
- infrastructure objectives
- autonomy objectives
- system completion goals

---

## What Does Not Belong Here

- individual code tasks
- raw ideas
- temporary notes
- daily execution logs
- one-off fixes
- low-priority improvements

---

## Hermes Usage

Hermes should use this file to understand the “why” behind the work.

Hermes should reference objectives before choosing next tasks.

Hermes should not treat objectives as permission to build anything loosely related.

Objectives define direction. Build phases define scope. Tasks define execution.

---

# Primary Objective

Build Simple Connect into a contractor-ready B2B automation platform centered around the Virtual Office Manager.

The VOM should help contractors and service businesses reduce front office workload by handling communication, booking, follow up, onboarding, and operational visibility.

---

# Objective 1: Contractor Demo Readiness

## Outcome

Create a stable, believable contractor-facing demo that clearly shows the VOM replacing manual office work.

## Success Criteria

- Vapi interaction is visible
- booking outcome is deterministic
- appointment confirmation is obvious
- dashboard activity is visible
- onboarding is visible as the next system step
- invoice follow-up is visible as future automation proof
- demo does not fail due to live voice instability

## Current Status

Active

---

# Objective 2: VOM Product Completion

## Outcome

Complete the VOM system enough to support early contractor client onboarding and usage.

## Success Criteria

- calls can be handled
- booking workflow exists
- onboarding workflow exists
- dashboard shows system activity
- invoice follow-up workflow exists
- human approval gates exist where needed
- production safety rules exist

## Current Status

In progress

# Objective 5: Production Safety

## Outcome

## Success Criteria

- production outreach is controlled
