# Vault Boundaries

## Purpose

This vault contains multiple operational domains.

Hermes must remain isolated to the SimpleConnect operational namespace unless explicitly instructed otherwise.

The goal is to prevent:
- operational contamination
- mixed priorities
- memory drift
- unrelated execution behavior
- cross-business confusion

---

## Approved Operational Scope

Hermes may autonomously operate inside:

SimpleConnect/

including:
- VOM
- B2B automation
- contractor demo workflows
- VPS infrastructure
- Codex execution
- Discord operations
- execution logging
- operational memory
- governance updates allowed by policy

---

## Restricted Areas

Hermes must not autonomously operate inside:

- HFM/
- Meetings/
- Companies/
- Strategic/
- Agents/

unless explicitly instructed by Dee.

---

## Namespace Rules

The SimpleConnect namespace should be treated as:
- the operational memory lane
- the execution coordination layer
- the source of truth for Simple Connect

Hermes should not:
- mix unrelated business context
- merge workflows between businesses
- reference HFM operational memory
- alter unrelated operational structures

---

## Operational Memory Rules

Hermes may:
- read approved governance files
- append operationally meaningful updates
- create concise execution summaries
- maintain operational continuity

Hermes may not:
- rewrite strategic direction
- alter governance without approval
- remove future phase visibility
- modify restricted namespaces

---

## Strategic Separation

Simple Connect and HFM are operationally separate systems.

Each system should maintain:
- separate governance
- separate execution memory
- separate priorities
- separate escalation logic
- separate operational context

---

## Rule

Hermes should treat:

SimpleConnect/

as the primary operational namespace and avoid operating outside that boundary unless explicitly instructed.
